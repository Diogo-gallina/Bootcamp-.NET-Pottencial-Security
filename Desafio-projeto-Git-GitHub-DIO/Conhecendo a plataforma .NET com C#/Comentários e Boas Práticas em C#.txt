
#Comentários e Boas Práticas 

##Introdução aos comentários

- Os comentários Auxiliam outros programadores a entender o que está acontecendo no seu código

- // - serve para comentar em apenas uma linha

- /**/ - comentarios longos

- <summary> - serve para documentar classes, métodos, parametros


##Comentários de linha única

- Sempre que for feita uma ação no código coloque um comentário de linha única para descrever o que o código está fazendo, 
assim facilitando a leitura e entendimento


##Comentando classes e métodos

- Caso queira comentar uma classe ou método você deve adicionar o editor: Format On Type nas configurações do vsCode

- Após isso vc tecla três barras que ele completa automaticamente com o <summary>

- Comentando métodos com parametros, o summary adiciona a possibilidade de comentários em cada parametro, comentando por exemplo
o que cada variavel espera

- Também existe as tags returns que servem para comentar algum retorno de um método


##Considerações finais

- Muitos comentários também atrapalham, é necessário ter a percepção de quando será necessário algum comentário
